Github Flow

Github Flow är en hjälpmedel när man jobbar i grupper, dela sin koder/filer eller lagra annat viktigt. Det kräver ett Github konto och programmet Git.

Branch

Brancher är något som används för att hålla koll på vad olika ändringar är. Det kommer alltid finnas en branch som heter master eller main, detta är huvudbranchen. Det är master/main branchen som alla filer mergas i för slutförning. En effektiv metod att använda vid grupp projekt för att skildra på varandra koder.

Add & Commit

För varje små ändringar, tillexempel bugg eller bortagning av fil, är det viktigt att "add" & "commit". Detta är för att hålla koll på ändringarna som sker i de olika brancher, se även till att vara detaljerad i commit meddelanden.

Pull Request

Pull request är delen av Github när man frågar ägaren av repot att kombinera sin branch med mainen. Det är en rescention för att se till att det är inget fel med koden och det är som andra delar av gruppen har tänkt.

Deploy & Merge

De två sista stegen är deploy och att merga branchen. Deploy är ett sista test för att se att koden i branchen fungerar såsom den ska och kan mergas med main branchen. Efter att branchen har klarat deploy testet kan användaren merga branchen ihop med main branchen och senare ta bort den tidigare.

Flexbox

Flexbox underlättar för programmeraren att göra responsiv. Att hemsidan är responsiv betyder att den anpassar sig efter användarens fönster format. Detta gör att programmering slipper använda mera komplicerade metoder som "float" eller "position". Flexbox funkar perfekt med användning av @media quieres, vilket används i syftet av anpassning till olika platformar. Man kan alltså använd flexbox och media quieres till att göra sin hemsida anpassningsbar till t.ex. mobil platformar.

Document Object Model

Doucment Object Model är en model som representerar,i detta fall HTML elementen som kan manipuleras med hjälp av program. Model består av ett träd fyllda med objekt och hur de är kopplade till varandra. Tillexempel är elementet body förälder till a och h1 elementet. Javascript får hjälp av DOM att lägga till/ändra och ta bort elementen som finns i det här trädet.

Fin kod

Att skriva en fin kod har som mål för att andra programmera kan göra den lätt att förstå. Detta täcker ett brett område ifrån hur variblar och funktioner skrivs till placering av mellanrum. Anledning till att regler vid namngivning av t.ex. funktioner och variablar är för andra att förstå vad de gör. Vid användning av funktioner och variablar använder vi oss av camelCase tillexempel.

Ramverk

Ramverk eller kodbibliotek specifikt inom programmering är ett hjälpmedel för att underlätta för front-end utvecklare. Man kan se det som ett bibliotek där deras källor (cascading style sheet funktinoner) bl.a. lånas i sin egen. Detta är en bra metod vid agila moment då gruppen har ont om tid.
